## 作業内容

### 1. フォルダ作成・リポジトリクローン

- `C:\ClaudeCode\ClaudeCodeRemoteTest_20260405` を作成
- GitHub リポジトリ `M-labo/ClaudeCodeRemoteTest_20260405` をクローン（空リポジトリ）

### 2. YOLOv7 Pose Estimationのセットアップ

```bash
git clone https://github.com/RizwanMunawar/yolov7-pose-estimation.git
```

- クローン後、`yolov7-pose-estimation/` を `.gitignore` に追加（GitHubにはアップロードしない）

```bash
pip install --upgrade pip
pip install -r requirements.txt
curl -L https://github.com/WongKinYiu/yolov7/releases/download/v0.1/yolov7-w6-pose.pt -o yolov7-w6-pose.pt
```

### 3. 入力動画の取得・変換

- Google Drive（gdown）でのダウンロードを試みたが、共有設定の制限により失敗
- Google Drive OAuth認証（`.gdrive_credentials.json`）を設定し、バックグラウンドで再試行 → `input_video.mov`（35MB）のダウンロードに成功
- `imageio-ffmpeg` を使い `.mov` → `.mp4` に変換 → `input_video.mp4`（8.9MB）

### 4. キーポイント検出の実行

以下の3つのバグをデバッグ・修正してから実行：

| バグ | 原因 | 修正内容 |
|------|------|---------|
| `AttributeError: 'list' object has no attribute 'seek'` | `--poseweights` の `nargs='+'` によりリスト型になり `strip_optimizer` に渡っていた | `opt.poseweights[0]` として文字列を取り出す |
| `_pickle.UnpicklingError: Weights only load failed` | PyTorch 2.6 で `torch.load` のデフォルトが `weights_only=True` に変更された | `utils/general.py` と `models/experimental.py` に `weights_only=False` を追加 |
| `RuntimeError: Sizes of tensors must match` | 入力動画が1080×1920でstride=64の倍数でなく、`letterbox` でサイズ不整合が発生 | letterbox の入力サイズを固定値 `960`（64の倍数）に変更 |

```bash
python pose-estimate.py --source ../input_video.mp4 --poseweights yolov7-w6-pose.pt --device cpu
```

### 5. GitHubへのアップロード
